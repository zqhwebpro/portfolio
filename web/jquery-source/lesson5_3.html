Server-side & $(getJSON)

 

The Scenario:

A client has come to you hoping that you could create a webpage that would loaded with server side data. The alert from before was an example of the webpage using a getJSON data request. The get request locates it and JSON... well, "JSON" is short for JavaScript Object Notation, it's an easy to understand, conscice storage code bank. Lesson explaination.

With "JSON" we are able to integrate "Ajax" (short for asynchronous javascript and xml) and output XML data exchanges. It requires a bit more then this example, this is just an introduction.

HTML:

For this lesson, only the head javascript script is required.

<head>
    <script src="lib/script/getjson.js"></script>
</head>

 

CSS:

No CSS is required of this lesson.



        
 


        
jQuery:


        
What I am doing in this series of code is is opening a getJSON request to load a url, a function is intiated with a selector of data, an alert is initiated of the function length from the request, and it is all closed up. It looks like this:

        
$(document).ready(function() { // doc ready
  $.getJSON('http://feeds.delicious.com/v2/json/tag/celebs?callback=?',  // open a getJSON requirest
    function(data) { // function of data
      alert('Loaded ' + data.length + ' item.'); // responds with data.length from .getJSON request
    }); // close data function
}); // close doc



        
 


        
Server Side getJSON:


        
Remember that the server side request loaded directly from server response to retreive data. JSON is a simple storage code method and AJAX synchronizes it with XML to open up functionallity and possibilities.
