Lesson 8 - CSS Positioning

Objective: Lesson 8 is about CSS position and how to manipulate to do exactly what you want. Positioning is extremely important for layout without it you will not be able to create the website of your dreams.

Static

By default items are placed in HTML using the static positioning system. Static positioning will order elements with the normal flow of the website and they are not affected by the top, bottom, right, left, or z-index properties. The primary use of setting position to static is to override another positioning element.

Fixed

Setting an item to a position of fixed will cause that specific item to ignore all of positioning and overlap everything. A fixed positioned element will also scroll with the screen. It stays exactly where you set it to stay.

Relative

Relative positioning also overlaps elements however, it instead searches for the parent element and places itself in terms of the parent elements. In it's spot it leaves it's area open and blank where it otherwise took up the position priority.

Absolute

Absolute positioning works like relative where it positions from the parent element. However, it does not leave it's spot behind. It is instead removed from normal flow and placed exactly to the set perimeters.

Float Positioning

Float Left & Right

Float breaks the flow of the static positioning by "floating" an item to the left or the right. It then takes up the space there and moves all other content around it. Floats are perfect for creating website with multiple colums that need seperated; it is the best way to go about creating that layout.

Clear:Both

In order to clear the float property you must use the clear:both; function. This will force the element to stop floating by clearing the direction of the float around a floated construct.

Overflow:Auto

The alternate to using the clear:both function is overflow:auto. By setting this property to a container all of the floated elements within the container then become children. This forces them to act as if they are in the normal flow of the document even through they are not. Around the container you can do whatever you would like with your positioning.

z-index Positioning

Outside of normal flow items might overlap one another. If this is your intent there is a way to prioritize what items are on top with z-index. By default all content on the website has a z-index of 0, if you set a positivie z-index it will appear above the elements, if you set a negative z-index: it will appear below the elements.
