Lesson 4 - Forms

Objective: Block and inline elements mastered, check. Now let me explain what forms are and describe the various properties a developer would need. You've seen forms all over the web. Put password here, what's your gender, type a comment here: all forms of forms. Pump yourself up for more conquesting as this learning adventure continues!

Form Tags:

<form>

the form tag is used to create forms. You can use a variety of types of forms to input data to be sent to the website and processed.

The above form is the form-ma-tron concept of forms. It lacks a method and get function so the data can not be processed. It still can be typed in to the box, so it is in fact a working form.

<fieldset>

fieldset tag is used to draw a box around an element in a form.

Check it out, our form has been field set.., TO THE MAX.

<legend>

legend defines the caption for the fieldset element.

Properties:

id

specifies a unique id for an element.

name

name is an input method asking for data identified by the developers form.

type

an input type including text, password, radio, checkbox, and submit.

for

used by the label tag to be a connector to a specific input id.

label

places an identifier for the optgroup tag.

value

a value of an input element.

checked

a pre-checked checkbox property

selected

pre-selected option on a list.
